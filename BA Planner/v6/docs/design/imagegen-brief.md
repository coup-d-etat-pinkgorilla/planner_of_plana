# BA Planner UI 목업 ImageGen 브리프

## 사용 방법

- Use case는 항상 `ui-mockup`으로 지정한다.
- 결과물은 콘셉트 아트가 아니라 실제 구현 가능한 Windows 데스크톱 제품 UI로 요청한다.
- 한 이미지에 전체 기능을 모두 넣지 않는다. 공통 방향을 유지한 채 화면별로 따로 생성한다.
- 기준 이미지는 16:9 전체 화면 캡처 구도로 만든다. 노트북 프레임, 책상, 손, 모니터 목업은 넣지 않는다.
- 학생 초상이나 재화 아이콘을 참조 이미지로 제공할 경우 역할을 `content reference`로 명시한다. 기존 화면 캡처는 `structure reference`, 원하는 분위기 이미지는 `style reference`로 구분한다.
- 이미지 안의 문구는 아래의 짧은 한국어 라벨만 사용한다. 임의의 긴 설명문이나 가짜 숫자를 과도하게 만들지 않는다.

## 프로그램의 용도

- Blue Archive의 학생 성장 상태와 인벤토리를 Windows 게임 화면에서 스캔하는 데스크톱 도구다.
- 스캔된 현재 상태를 학생별로 탐색하고 성장 목표를 설정한다.
- 현재 상태에서 목표까지 필요한 재화를 계산한다.
- 총 필요량, 현재 보유량, 실제 부족량을 구분해 우선순위를 판단한다.
- 전술대항전 기록과 상대 덱 대응 정보를 별도 작업공간에서 관리한다.
- 핵심 가치는 장식적인 게임 대시보드가 아니라 정확한 상태 확인과 성장 계획 의사결정이다.

## 핵심 사용자 흐름

목업은 다음 순서를 유지한다.

1. `프로필 및 게임 창 연결` 상태를 확인한다.
2. `학생 / 자원 / 아이템 / 장비` 중 스캔 대상을 선택한다.
3. 스캔 진행률, 현재 인식 대상, 캡처 미리보기, 중지 동작을 확인한다.
4. 학생을 검색하고 현재 성장 상태를 확인한다.
5. 학생을 플랜에 추가하고 목표 레벨, 성급, 무기, 스킬, 장비를 설정한다.
6. 선택 학생 또는 여러 학생의 필요 재화를 계산한다.
7. 보유량과 비교해 부족 재화와 영향 학생을 확인한다.

화면을 재구성해도 사용자의 인지 흐름은 `연결 및 스캔 → 학생 확인 → 목표 설정 → 부족 판단`으로 읽혀야 한다.

## 반드시 유지해야 할 기능과 정보

### 데이터 의미

- `현재`는 스캔으로 확인한 실제 보유 상태다.
- `목표`는 사용자가 직접 정한 성장 목표다.
- `필요`는 현재에서 목표까지 필요한 총량이다.
- `보유`는 스캔된 인벤토리 수량이다.
- `부족`은 필요량과 보유량을 비교한 결과다.
- 현재, 목표, 필요, 보유, 부족을 하나의 숫자나 같은 색으로 합치지 않는다.
- 인벤토리를 아직 스캔하지 않은 상태와 보유량 0을 다르게 표현한다.
- 미보유 학생은 학생 목록에는 표시할 수 있지만 가상의 현재 성장값을 보여주지 않는다.

### 스캔

- 현재 프로필과 선택된 게임 창을 항상 확인할 수 있어야 한다.
- 게임 창 연결 여부와 스캔 가능 상태가 명확해야 한다.
- 학생, 현재 학생, 자원, 아이템, 장비 스캔 진입점을 유지한다.
- 스캔 진행률, 현재 처리 대상, 상태 메시지, 중지 버튼을 유지한다.
- 캡처 미리보기는 16:9 비율을 유지한다.
- 오류, 중지 요청, 완료, 검토 필요 상태가 서로 구분되어야 한다.

### 학생 탐색

- 학생 카드 그리드와 선택 학생 상세를 유지한다.
- 이름·ID·태그 검색을 유지한다.
- 정렬, 미보유 학생 표시, JP 전용 숨김, 상세 필터를 유지한다.
- 카드에는 학생 초상, 이름, 보유 여부를 빠르게 구분할 수 있는 정보가 필요하다.
- 상세에는 레벨, 성급, 전용무기, 스킬, 장비, 공격/방어 타입을 보여준다.
- 학생 상세에서 플랜에 추가할 수 있어야 한다.

### 플랜과 필요 재화

- 계획 학생 목록과 선택 학생 목표 편집기를 유지한다.
- 목표 레벨, 성급, 전용무기, 스킬, 장비, 능력개방 입력을 유지한다.
- 목표값을 바꾸면 선택 학생의 필요 재화를 바로 확인할 수 있어야 한다.
- 여러 학생을 계산 범위에 추가하고 합산할 수 있어야 한다.
- 계획이 없는 학생에 대해 레벨·장비·스킬 계산 포함 여부를 선택할 수 있어야 한다.
- 계획 학생의 수동 순서가 우선순위라는 의미를 유지한다.
- 자동 저장 상태 또는 마지막 저장 상태를 확인할 수 있게 표현한다.

### 인벤토리

- 장비와 아이템을 구분한다.
- 카테고리, 재화명, 아이콘, 보유량을 탐색할 수 있어야 한다.
- 계획 부족과 전체 육성 부족을 서로 다른 범위로 표현한다.
- 부족 재화를 선택하면 영향을 받는 학생과 요구량을 확인할 수 있어야 한다.
- 부족 비율, 충족률, 우선순위를 빠르게 비교할 수 있어야 한다.

## 중요도가 높은 화면 요소

첫 시선이 다음 순서로 이동하도록 구성한다.

1. 현재 화면의 주 작업과 연결 상태
2. 선택 학생 또는 선택 재화의 핵심 정보
3. 주요 행동 버튼 하나
4. 부족 여부 또는 진행 상태
5. 상세 수치와 보조 필터

다음 요소는 시각적으로 가장 높은 우선순위를 갖는다.

- 프로필과 게임 창의 연결 상태
- 현재 실행 가능한 대표 스캔 행동
- 스캔 진행률과 중지 버튼
- 학생 검색과 학생 카드 그리드
- 선택 학생의 현재값과 목표값 비교
- `플랜에 추가` 또는 `목표 저장됨` 상태
- 부족 재화의 이름, 부족량, 충족률
- 계산 범위에 포함된 학생 수
- 오류, 검토 필요, 인벤토리 미스캔 경고

주요 행동은 채워진 버튼 또는 강한 대비로 표현한다. 일반 버튼, 선택 상태, 위험 버튼을 모두 같은 분홍 외곽선으로 표현하지 않는다.

## 중요도가 낮은 요소

다음 요소는 접기, 보조 메뉴, 컨텍스트 액션, 별도 설정 화면으로 내려도 된다.

- 전체 로그 원문과 기술 진단 메시지
- 빠른 스캔 기준 목록 편집
- 입력 테스트와 캡처 영역 테스트
- JP 전용 숨김 같은 고급 필터
- 상세 메타데이터 필터 그룹
- 미계획 학생의 세부 계산 옵션
- 인벤토리의 전체 육성 장기 압력 분석
- 전술대항전 줄임말 사전
- Excel 가져오기와 이름 일괄 수정
- 프로필 생성·삭제와 창 목록 새로고침
- 현재 프로필 데이터 삭제

중요도가 낮다는 것은 제거를 뜻하지 않는다. 기본 화면의 주 작업과 경쟁하지 않게 배치한다.

## 기준 해상도

- 기준 캔버스: `1920×1080`, 16:9, Windows 100% 배율
- 전체 앱 화면을 정면에서 보여주는 실제 제품 스크린샷 구도
- 외곽 여백은 16~24px 범위
- 본문은 12px 또는 8px 기반 간격 체계를 사용
- 본문 기본 글자는 14~16px로 읽을 수 있게 표현
- 보조 글자는 12px 미만으로 과도하게 축소하지 않음
- 클릭 목표는 최소 32~40px 높이로 보이게 구성
- 화면 전체가 데이터로 채워져도 주요 작업이 한눈에 보여야 함

## 하위 검증 해상도

- `1536×864`: 일반 노트북과 Windows 125% 배율을 가정한 축소 검증
- `1366×768`: 최소 지원 후보, 3열을 그대로 압축하지 말고 2열 또는 단계형 구조로 전환
- `2560×1440`: 본문 최대 폭과 과도한 빈 공간 검증
- `3440×1440`: 울트라와이드에서 콘텐츠가 끝없이 늘어나지 않도록 최대 폭 또는 보조 패널 전략 검증
- 서로 다른 DPI의 듀얼 모니터에서도 Qt 본창과 스캔 오버레이가 같은 시각 크기로 느껴져야 함

작은 해상도 목업은 전체 UI를 단순 비례 축소하지 않는다. 보조 패널을 접고, 열 수를 줄이고, 상세를 드로어 또는 별도 단계로 이동한다.

## 원하는 분위기

- 전문적인 Windows 데스크톱 플래너
- 차분하고 정밀한 전술 작업공간
- 게임 팬 도구의 친근함과 생산성 도구의 신뢰감을 함께 가짐
- 학생 초상이 데이터보다 장식적으로 튀지 않고 탐색 기준으로 기능함
- 어두운 남색 바탕에 제한된 분홍 포인트와 밝은 하늘색 상태 색
- 단단한 그리드, 명확한 정렬, 절제된 곡률, 얕은 레이어 깊이
- 현재값과 목표값의 비교가 자연스럽게 읽히는 구조
- 부족과 오류는 분명하지만 불안감을 과장하지 않는 표현
- 긴 사용 시간에도 피로가 적은 본문용 산세리프 타이포그래피
- 숫자는 자릿수 비교가 쉬운 tabular numerals 스타일
- Blue Archive의 경쾌한 색감을 연상시키되 공식 게임 UI를 복제하지 않는 독립적인 제품 디자인

권장 팔레트 출발점:

- 앱 배경: 매우 어두운 네이비 또는 블루블랙
- 기본 패널: 배경보다 한 단계 밝은 저채도 네이비
- 주요 포인트: 제한적으로 사용하는 핑크 또는 마젠타
- 연결/정상 상태: 밝은 시안 또는 하늘색
- 충분/완료 상태: 청록 또는 녹색
- 부족/경고 상태: 코랄 또는 주황
- 오류/삭제 상태: 명확한 빨강
- 본문 텍스트: 거의 흰색
- 보조 텍스트: 충분한 대비의 블루그레이

## 피하고 싶은 스타일

- 공식 Blue Archive 게임 UI를 그대로 복제한 화면
- 모바일 가챠 게임 홈 화면처럼 장식이 많은 인터페이스
- 네온 사이버펑크, 과도한 발광, 홀로그램, 유리 효과
- 모든 패널에 강한 그라데이션과 그림자를 사용한 스타일
- 둥근 카드가 겹겹이 중첩된 소프트 SaaS 대시보드
- 지나치게 넓은 여백과 큰 카드만 있는 저밀도 마케팅 대시보드
- 데이터 표를 스프레드시트처럼 회색 선으로만 구성한 무표정한 엔터프라이즈 UI
- 모든 버튼과 선택 상태가 같은 분홍 테두리인 스타일
- 작은 회색 글씨에 중요한 상태를 숨기는 표현
- 3열을 작은 화면에서 그대로 축소하는 레이아웃
- 학생 초상과 재화 아이콘을 배경 장식으로 크게 사용하는 구성
- 불필요한 캐릭터 일러스트, 마스코트, 말풍선이 주 작업을 가리는 구성
- 가짜 3D 창, 원근이 들어간 모니터 목업, 비스듬한 화면
- 읽을 수 없는 장식 문자, 임의의 영어 메뉴, 가짜 로고, 워터마크

## 전문 도구에 필요한 정보 밀도

- 정보 밀도는 `중간보다 높음`으로 유지한다.
- 한 화면에서 선택 대상, 현재 상태, 다음 행동, 결과를 함께 판단할 수 있어야 한다.
- 모든 정보를 동시에 펼치지 말고 핵심 60%, 보조 30%, 고급 10%의 위계로 구성한다.
- 반복 목록은 행 높이를 일정하게 하고 숫자 열을 정렬한다.
- 카드보다 리스트가 비교에 유리한 정보는 리스트나 표 형태를 사용한다.
- 학생 탐색은 초상 카드가 적합하고, 재화 비교는 정렬된 행이 적합하다.
- 요약 숫자는 크게 만들되 전체 화면을 차지하는 KPI 카드로 만들지 않는다.
- 패널 경계보다 제목, 정렬, 간격, 배경 단계로 그룹을 구분한다.
- 긴 설명 대신 짧은 상태 문구, 배지, 범위 라벨, 툴팁을 사용한다.

## 핵심 UI 영역의 역할

### 결과 목록

- 결과 목록은 탐색과 비교를 위한 작업 영역이다.
- 학생 결과는 초상, 이름, 보유 여부, 핵심 성장 상태를 빠르게 훑을 수 있는 카드 그리드로 표현한다.
- 재화 결과는 아이콘, 이름, 필요, 보유, 부족, 충족률을 열 정렬한 고밀도 목록으로 표현한다.
- 선택 상태, 계산 범위 포함 상태, 플랜 포함 상태를 서로 다른 시각 언어로 구분한다.
- 목록 상단에는 현재 결과 수, 적용 필터, 정렬 기준을 짧게 표시한다.
- 빈 결과는 이유와 다음 행동을 한 문장으로 안내한다.

### 상태 표시

- 상태 표시는 사용자가 지금 안전하게 행동할 수 있는지를 판단하게 한다.
- 프로필, 게임 창, 연결, 스캔 준비, 진행, 완료, 오류를 한 영역에서 일관되게 보여준다.
- 정상 상태는 조용하게, 경고와 오류는 명확하게 표현한다.
- 색상만 사용하지 않고 아이콘, 라벨, 문장, 진행률을 함께 사용한다.
- 자동 저장은 `저장됨`, `저장 중`, `저장 실패` 세 상태를 짧게 보여준다.
- 기술 로그는 기본 상태 표시 아래의 접을 수 있는 보조 영역으로 둔다.

### 캡처 미리보기

- 캡처 미리보기는 스캔 신뢰성을 확인하는 증거 영역이다.
- 16:9 게임 화면 비율을 유지하고 왜곡하거나 장식 프레임으로 가리지 않는다.
- 현재 인식 중인 영역 또는 대상을 미묘한 오버레이로 표시한다.
- 진행률과 현재 학생/아이템 정보는 미리보기 위를 과도하게 가리지 않는다.
- 미리보기 자체가 없는 상태, 연결 끊김, 비율 불일치 상태를 구분한다.
- 일반 사용자에게는 작은 확인 창으로, 진단 모드에서는 확장 가능한 구조로 표현한다.

### 설정 패널

- 설정 패널은 주 작업공간이 아니라 연결과 관리의 보조 영역이다.
- 현재 프로필과 선택된 게임 창을 가장 먼저 보여준다.
- `프로필 적용`, `새 프로필`, `창 선택`을 일반 관리 기능으로 묶는다.
- `현재 프로필 데이터 삭제`는 별도 위험 영역과 확인 단계로 분리한다.
- 입력 테스트, 빠른 스캔 기준 편집, 상세 진단은 고급 설정 아래에 둔다.
- 설정의 변경 결과가 스캔 탭 상태에 즉시 반영된다는 관계를 보여준다.

## 생성 결과 평가 체크리스트

- 주 작업을 3초 안에 설명할 수 있는가?
- 프로필과 게임 창 연결 상태가 보이는가?
- 현재값과 목표값이 혼동되지 않는가?
- 필요, 보유, 부족을 따로 읽을 수 있는가?
- 주요 행동과 보조 행동과 위험 행동이 다르게 보이는가?
- 학생 카드가 장식이 아니라 탐색 수단으로 기능하는가?
- 재화 비교는 카드보다 정렬된 목록으로 표현되었는가?
- 캡처 미리보기가 16:9이고 상태 정보를 가리지 않는가?
- 본문이 1366×768에서 단순 축소되지 않고 재배치될 수 있는 구조인가?
- 고급 설정이 기본 작업과 경쟁하지 않는가?
- 공식 게임 UI 복제나 모바일 가챠 UI처럼 보이지 않는가?
- 실제 Qt/Tk 데스크톱 앱으로 구현 가능한가?
- 불필요한 로고, 워터마크, 가짜 영문 메뉴가 없는가?

한 번의 생성에서 모든 조건을 고치려 하지 않는다. 첫 결과에서는 구조와 정보 위계를 평가하고, 이후 반복에서는 `주요 행동 강조`, `목록 밀도`, `현재/목표 비교`, `작은 화면 대응`처럼 한 가지 변화만 요청한다.
