# UI Component Studio

## 스캔 런타임 섹션

섹션 그리드에는 스캔 화면의 실제 작업 단위가 각각 독립 항목으로 표시됩니다.

- `scanDebugSection`: 진행 메시지와 디버그 로그 출력
- `scanMirrorSection`: 현재 스캔 결과 미러 컨테이너
- `scanStudentCard`: 학생 스캔 결과 미러
- `scanInventoryCard`: 아이템·장비 그리드 결과 미러
- `scanWorkSection`: 진행률과 캡처 미리보기를 함께 감싸는 우측 작업 열
- `scanProgressSection`: 진행률과 스캔 중지 버튼 영역
- `scanPreviewSection`: 검은 16:9 캡처 미리보기 영역
- `scanResultSection`: 스캔 완료 뒤 전환되는 결과 영역

이 항목들의 실제 크기 미리보기는 디자인 모드의 메인 UI에서 해당 위젯을 분리하므로, 하부 트리와 main 좌표를 그대로 유지합니다. `Q`와 `W`는 실제 진입 방향에 맞춰 좌우 intro/outro를 재생합니다.

섹션 그리드의 카드도 동일한 디자인 모드 런타임 트리에서 만든 스냅샷을 사용합니다. 스냅샷은 카드의 썸네일 캔버스 안에 비율 유지(`KeepAspectRatio`)로 축소하며, 실제 UI에서의 가로세로 비율을 바꾸지 않습니다. 카드와 원본 비율이 다르면 남는 공간을 여백으로 두고 섹션을 늘이거나 찌그러뜨리지 않습니다.

## 상태 머신 탭

`상태 머신` 탭에서 메인 탭을 고르면 시작 상태, 섹션 부모-자식 호출 구조, 상태 전환 트리거, 메인 탭 진입·이탈 구조를 함께 볼 수 있습니다. `필요 재화`와 `통계`처럼 기본 탭 바에서 숨겨진 탭도 포함됩니다. 이 화면은 구조 확인용이며 실제 기능이나 데이터를 실행하지 않습니다.

상태 머신 탭은 다음 세 보기로 구성됩니다.

- `Markdown 문서`: `docs/ui-behavior/`의 탭별 Markdown을 실제 Mermaid로 렌더링
- `섹션 전환표`: stable ID, 출발 섹션, 전환 버튼/이벤트, guard, action, 도착 섹션, 보이는 결과
- `연결 감사`: 섹션 전환 후보 signal과 전환 계약의 대응 여부

Markdown 문서는 `전체 화면`, `홈`, `학생부`, `계획`, `필요 재화`, `인벤토리`, `전술대항전`, `통계` 단위로 나뉩니다. 문서 안에서는 동일한 섹션 노드를 재사용하는 Mermaid `flowchart`를 사용하므로 전환마다 노드를 반복하는 일대일 행 형태가 아닙니다. UCS는 QtWebEngine과 로컬 Mermaid 번들을 이용해 이 파일을 인터넷 연결 없이 렌더링합니다. `Markdown 파일 열기`, `문서 새로고침`, `경로 복사`도 지원합니다.

연결 감사는 각 탭 소스에서 signal 연결을 조사하되, 스택 페이지·다이얼로그·메인 탭·큰 섹션을 바꾸는 연결만 남깁니다. 새로고침, 복사, 삭제, 검색어 입력처럼 현재 섹션 안에서 끝나는 지엽적인 동작은 그래프와 감사에서 제외됩니다. 전환 계약과 대응되지 않는 섹션 연결 후보는 `검토 필요`와 붉은 배경으로 표시됩니다. `검토 필요만`을 선택하면 동적 연결 또는 소스 확인이 필요한 전환만 모아서 볼 수 있습니다.

모든 stable ID는 표시 문자열과 분리된 영문 의미 ID를 사용합니다. 새 버튼이 섹션 이동을 만들거나 스택 페이지 전환이 추가됐는데 전이 계약이 없으면 연결 감사에 후보로 나타나므로, 큰 화면 흐름이 조용히 누락되는 것을 방지할 수 있습니다.

## 대화로 UI 편집을 지시하는 방법

섹션이나 위젯 카드를 선택하면 오른쪽의 `선택 섹션 하부 트리`에 실제 클래스, `objectName`, 표시 텍스트가 나타납니다. 수정하려는 요소를 고르고 `대화용 지시 경로 복사`를 누른 뒤, 복사된 문장의 `[변경할 내용]`을 원하는 내용으로 바꾸면 됩니다.

```text
UCS > 섹션 > section:scanStudentMirrorSection > .../QLabel#detailBigValue:
학생 레벨 숫자를 4px 키우고 accent 색상을 사용. 스캔 진행 상태와 완료 상태 모두 적용.
```

육안 설명만 사용하는 것보다 `안정적인 트리 경로 + 적용할 상태 + 변경값`을 함께 주는 방식이 안전합니다. 위치 관계가 핵심이면 미리보기 스크린샷과 기준 모서리·간격을 덧붙이는 것이 가장 정확합니다. 수정 후 실제 크기 미리보기의 정지 상태를 확인하고 `Q`/`W`로 양쪽 애니메이션까지 검토합니다.

UI Component Studio(UCS)는 BA Planner 컴포넌트의 **디자인 전용 갤러리**입니다.
실제 `StudentViewerWindow`나 탭, 데이터 목록을 복제하지 않습니다.

## 실행

```powershell
py -3.11 -m tools.ui_component_studio_quick
# 또는
run_ui_component_studio.bat
```

기본 UCS는 Qt Quick 고정 1920×1080 논리 캔버스와 가상 목록을 사용한다. 디자인 명세
JSON만 읽으므로 실제 `StudentViewerWindow`나 대형 QWidget 갤러리 트리를 만들지 않는다.
Quick UCS의 안정 대상에는 주요 패널, 공통 컨트롤과 `PlannerPopup` 기반 대화상자·드롭다운
오버레이, 역할별 반복 표면과 초상·아이콘·배지·통계 막대·요약 카드 요소가 포함된다. 오버레이 편집기는 실제 페이지를 열지 않고
표면, 테두리, 패딩과 모달 스크림을 독립적으로 미리 본다. 반복 표면 편집기는 NORMAL,
ALTERNATE, SELECTED 상태를 동시에 표시해 목록 밀도와 선택 위계를 비교한다.
내부 요소 편집기는 자동 크기 축을 `0`으로 유지하면서 표면과 불투명도를 독립적으로 조정한다.
페이지에서 보이는 구분선도 `elements/divider` 항목으로 편집하며 직접 Rectangle 예외를 두지 않는다.

QML 패널 항목을 선택하면 중앙 미리보기는 단독 `PlannerSurface` 샘플 대신 데이터 로드를 끈
실제 `Main.qml`을 사용한다. UCS는 선택한 `componentKey`의 main 좌표와 논리 크기를 찾아 해당
영역만 균일 확대하므로, 섹션 내부 버튼·텍스트·이미지·레이아웃과 종횡비가 Viewer와 같다.
오른쪽 `섹션 객체 트리`는 그 실제 QML 자식 트리를 펼쳐 표시하며 섹션 루트 경로 또는 각
아이템 경로를 클립보드에 복사할 수 있다.

`미리보기 창`은 별도 16:9 창의 1920×1080 논리 캔버스에 선택 섹션을 Main과 동일한 좌표·크기로
클립해 표시한다. 창을 1280×720이나 960×540으로 줄여도 캔버스 전체에 하나의 scale만 적용하므로
섹션을 창에 맞춰 늘이거나 찌그러뜨리지 않는다. 미리보기 표면 위의 입력 가드는 실제 페이지
동작을 차단한다.
기존 상태 머신 감사·실제 QWidget 스냅샷 도구가 필요한 경우에는 다음 호환 명령을 사용한다.

```powershell
py -3.11 -m tools.ui_component_studio
```

## UCS를 종료하지 않고 소스 새로고침

상단의 `소스 새로고침`을 누르면 실행 중인 UCS 창은 유지하면서 GUI 시각 모듈을 다시 불러오고,
갤러리 카드와 선택 트리를 재생성합니다. 실제 크기 미리보기가 열려 있었다면 같은 항목으로 다시 엽니다.
따라서 `gui/triangle_texture.py`, `gui/diagonal_scroll_list.py`, `gui/viewer_shared.py`,
`gui/viewer_components/`, `gui/viewer_app_qt.py`의 변경을 UCS 재시작 없이 확인할 수 있습니다.
`homeConnectionSection` 갤러리 카드는 별도 복제 샘플이 아니라 실제 `HomeTabComponent` 빌더로
다시 생성되므로, 목록 마진·버튼 형상·행 텍스처와 같은 런타임 변경도 갤러리에 반영됩니다.

이 기능은 GUI 시각 모듈을 갱신한 뒤 실행 중인 `tools/ui_component_studio.py`의 샘플 팩토리도 같은
모듈 객체에서 다시 불러옵니다. 따라서 UCS 도구 안의 섹션 샘플 라우팅 변경도 재시작 없이 반영됩니다.
Qt 플러그인이나 Python 환경을 변경한 경우에만 UCS를 재시작해야 합니다. 새로고침 실패 사유는 툴바
아래 상태 문구에 표시되며, 기존 디자인 명세와 draft는 유지됩니다.

## Component Gallery 분류

UCS의 Component Gallery는 `위젯`과 `섹션` 탭으로 나뉩니다.

두 탭 모두 2열 그리드로 표시됩니다. 홈 계열 섹션 타일은 내부 대각선 메뉴와 설정 행이
납작해지지 않도록 일반 위젯보다 큰 최소 높이를 사용합니다.

- `위젯`: Label, Button, 입력 필드, 콤보박스, 진행 막대처럼 독립적으로 동작하는 컨트롤을 표시합니다.
- `섹션`: 홈 메뉴·연결·스캔·설정 섹션, 스캔 복합 헤더, 플랜 편집 카드처럼 여러 위젯을 감싸는 시각적 표면을 표시합니다.

위젯 탭의 이미지 샘플은 다음 프로젝트 자산을 고정 레퍼런스로 사용합니다.

- 정사각 이미지와 학생 Portrait: `templates/students_portraits/seia.png`
- 재화 아이콘: `templates/icons/ooparts/Item_Icon_Material_Nebra_2.png`

정사각 이미지는 세이아 portrait를 중앙 기준으로 잘라 표시하고, Portrait와 재화
아이콘은 원본 종횡비를 유지합니다. 이 레퍼런스는 이미지가 들어가는 신규 카드와
행의 크기·크롭·내부 여백을 비교하는 기준입니다.

섹션 샘플은 실제 Viewer가 사용하는 `HomeGlassSection`, `DiagonalScanFrame`,
`SlantedScanHeader`, `PlanEditorSectionCard` 등의 표면 클래스로 렌더링합니다.
따라서 절단면, 확장 웨지, 둥근 모서리, 복합 헤더 연결 형태를 실제 Planner를
복제하거나 기능 데이터를 불러오지 않고 확인할 수 있습니다.

기존 디자인 명세에 저장된 selector도 이름에 `Section`, `Panel`, `Band`,
`Header`, `Dashboard`, `Workspace`가 포함되면 섹션 탭으로 자동 분류됩니다.

## 계획 기능 위젯 카탈로그

Gallery에는 실제 기능 연결 전에 디자인을 검수할 수 있도록 다음 계약 샘플이
포함됩니다.

- 학생: 카드, 압축 행, 필터 바, 그룹 선택기
- 목표: 목표 편집기, 프리셋, 마일스톤, 일괄 적용
- 계획: 상태 배지, 그룹 카드, 우선순위 목록, 시나리오 비교
- 재화: 요구량 행·표, 충족률 막대, 사용처, 카테고리 요약
- 통계: KPI, 가로 막대, 히스토그램, 누적 막대, 히트맵, 산점도, 추이선
- 분석: 현재/목표 비교, 증분 비용, 병목, 충돌 경고
- 데이터 품질: 스캔 상태, 최신성 배지, 이상 변화 목록
- 공통 조작: 필터 칩, 정렬 메뉴, 보기 전환, 상세 드로어, 빈 상태

이 샘플들은 대표 데이터를 사용한 디자인 전용 컴포넌트입니다. Planner의 계획
저장, 재화 계산, 필터 상태나 스캔 결과에는 연결되지 않습니다.

## 편집 범위

갤러리 카드에는 QLabel, QPushButton, QFrame, QLineEdit, QComboBox,
QProgressBar, QPlainTextEdit의 독립 샘플이 표시됩니다. 기존 selector 기반 디자인이
있으면 별도 카드로 함께 표시됩니다.

UCS에서 변경할 수 있는 값은 다음과 같습니다.

- 공통 위젯 종류별 QSS
- 기존 selector 대상의 QSS와 대각선 형태
- 공통 색상 팔레트

UCS는 위치, 크기, layout, 표시/활성 상태, 새 위젯 삽입, 애니메이션, 디버그 화면
전환을 변경하거나 실행하지 않습니다. v1 명세의 구조 관련 값은 마이그레이션 이력을
위해 JSON에 보존되지만 Planner 런타임에서는 적용되지 않습니다.

## 실제 크기 미리보기

갤러리 카드를 선택한 뒤 `선택 항목 미리보기`를 누르면 Planner 기준 크기(1920×1080)의 별도
창이 열립니다. 창 배경은 회색이며 선택한 컴포넌트는 실제 main의 정착 좌표에 표시됩니다.
예를 들어 홈 메뉴는 왼쪽 슬롯, 설정은 중앙 슬롯, 연결·스캔은 오른쪽 슬롯, 스캔 헤더는
탭 콘텐츠 상단의 실제 좌표를 사용합니다. 섹션을 선택하면 루트
표면뿐 아니라 그 아래의 버튼, 텍스트, 텍스처, 그라디언트, 그림자 등 전체 샘플 트리를 함께
렌더링합니다. 미리보기는 hover·enter·leave 같은 시각 피드백을 재생하지만 클릭·더블클릭·
컨텍스트 메뉴는 입력 가드가 차단하므로 실제 기능에는 연결되지 않습니다.

- `Q`: 선택한 컴포넌트의 intro 애니메이션을 재생합니다.
- `W`: 선택한 컴포넌트의 outro 애니메이션을 재생합니다.

두 키는 창 범위 단축키이므로 콤보박스나 입력 컨트롤에 포커스가 있어도 동작합니다.

홈 메뉴 섹션은 실제 Viewer와 같은 `HomeGlassSection`, `HomeMenuRowsWidget`,
`ParallelogramActionButton` 조합과 프로젝트 텍스처 자산을 사용합니다.
홈 설정은 설정 헤더·계정 관리·창 연결 해제·지원의 네 하위 표면을 모두 렌더링합니다.
홈 연결과 스캔도 각각 창 후보 목록과 대각선 작업 버튼 전체를 표시하며, 아이템 범위와
자원 입력 섹션도 독립 갤러리 항목으로 선택할 수 있습니다.
연결·설정의 실제 크기 미리보기는 별도 재현 샘플 대신 디자인 모드 `StudentViewerWindow`가
생성한 실제 섹션 트리와 main QSS를 그대로 사용합니다.
실제 runtime selector가 없는 카탈로그 전용 샘플은 중앙 배치하지 않고 main 탭 콘텐츠 원점에
배치합니다.

## 런타임 성능

v1 런타임은 명세 대상이 몇 개뿐이어도 실제 Viewer의 모든 QWidget을 두 번 순회해
selector index를 만들었습니다. v2는 명세에 기록된 selector만 부모-자식 경로로
직접 탐색합니다. 위젯 종류별 QSS도 하나의 root stylesheet 레이어로 적용하므로
자식 트리를 Python에서 순회하지 않습니다.

## 저장

`디자인 저장`은 `gui/ui_design_spec.json`에 version 2 형식으로 저장합니다.
편집 중 값은 `debug/ui_studio_design_draft.json`에 임시 저장됩니다.

## 다중 영역 실제 크기 미리보기

실제 크기 미리보기 창이 열린 상태에서 갤러리의 다른 항목을 선택하고 `선택 항목 미리보기`를 누르면 기존 창에 새 영역이 누적됩니다. 좌표가 겹치는 서로 다른 영역도 함께 표시하며, 가장 최근에 추가한 영역을 최상단에 그립니다. 동일한 UCS 항목 ID를 다시 요청한 경우에만 중복 추가하지 않습니다.

`Q`와 `W` 단축키는 미리보기 창에 누적된 모든 영역의 intro/outro 애니메이션을 동시에 재생합니다.
